# Nicholas B. King — academic site

This repo is a static personal/academic website for Nicholas B. King (Associate
Professor, Department of Equity, Ethics, and Policy / Max Bell School of Public
Policy, McGill University; Associate Director, Trottier Institute for Science
and Public Policy). It is deployed via GitHub Pages from the `main` branch,
root folder.

## How the site is built

There is no Hugo/Jekyll/JS build toolchain. The site is generated by a small
Python script:

- `data.py` — all structured content: publications (`JOURNAL_ARTICLES`,
  `BOOK_CHAPTERS`, `OTHER_PUBLICATIONS`), talks (`INVITED_TALKS`,
  `CONFERENCE_PRESENTATIONS`, `PUBLIC_LECTURES`), `MEDIA`, `RESEARCH_AREAS`,
  `SOFTWARE`, `TEACHING_COURSES`, `TEACHING_AWARDS`, `APPOINTMENTS_CURRENT`,
  `APPOINTMENTS_PREVIOUS`, `SELECTED_GRANTS`, plus link-lookup dicts
  `PUBLICATION_LINKS` (keyed by exact publication HTML string) and
  `TALK_LINKS` (keyed by `(date, title)` tuple) that add "Read online" /
  clickable-title links where a URL is known.
- `build.py` — templates everything with Python `.format()` strings (no
  Jinja) and writes the final static HTML files to the repo root:
  `index.html`, `publications.html`, `talks.html`, `software.html`,
  `teaching.html`, `media.html`, `contact.html`.
- `assets/css/style.css`, `assets/js/main.js` — hand-written CSS/JS, no
  frameworks, no build step.
- `images/` — headshot and favicon files. `files/King_CV_2026.pdf` — CV for
  the "CV" download button.

**To make any content change: edit `data.py` and/or `build.py`, then run
`python3 build.py` from the repo root to regenerate the HTML files, then
commit everything (including the regenerated `.html` files — they are
checked in, not built by GitHub Pages).**

There is no `bio.html` / "Bio & CV" page — it was deliberately removed. Bio
content and CV/Contact/Google Scholar links live on the homepage instead
(see below).

## Homepage structure (index.html / build_index() in build.py)

Two-column "hero" layout modeled on https://georgeyean.github.io/ (photo,
position, about on the left; research boxes on the right) and
https://brandonstewart.org/ (pill-shaped CV/Contact/Google Scholar buttons):

- `hero-left`: circular photo, `<h1>` name, two `position-block`s (Associate
  Professor / Associate Director — titles bolded, verbatim per the user's
  specified text), a `button-row` with three `btn btn-secondary` buttons
  (CV → `files/King_CV_2026.pdf`, Contact → mailto, Google Scholar →
  https://scholar.google.ca/citations?user=LWgI4HgAAAAJ&hl=en), then an
  "About Me" section with a verbatim biography paragraph.
- `hero-right`: "Research" heading followed by `HOMEPAGE_RESEARCH` boxes
  (`.research-box` divs), one per featured paper, each with title, authors,
  venue, a short description, and a link. As of the last update there are
  three: the DeBono et al. 2025 InfluenceMapper paper, the Dimitris et al.
  2021 BMJ Global Health authorship paper, and King 2020 "Harm Reduction: A
  Misnomer." Descriptions were supplied verbatim by the site owner — do not
  paraphrase them further without being asked.
- No accordion, no "Recent Highlights" section, no borders on plain lists —
  all removed per explicit request. If asked to add a "Research Areas" list
  back, use a plain `<ul class="plain-list-flush">`, not an accordion.

## Design conventions (do not change without being asked)

- **No custom web fonts.** System font stack only
  (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial,
  sans-serif`).
- **Color palette**: slightly bluish slate-gray, not cream/warm tones. See
  `:root` in `assets/css/style.css` (`--color-bg: #eef1f4`, etc.). If asked
  to adjust the palette, keep the same "sleek, minimalist" character.
- **Header/nav**: left-justified (brand + nav links together on the left,
  not split with `space-between`), with thin vertical bar dividers between
  each nav item (see `.nav-links li:not(:first-child)::before` in the CSS).
  Modeled on https://georgeyean.github.io/. Brand text is intentionally
  large (`font-size: 1.55rem`).
- **Buttons**: `.btn` is the shared pill shape; `.btn-secondary` (outline,
  no fill) is the default style used for all three homepage buttons — there
  is deliberately no filled/dark `.btn-primary` button on the page anymore.
- **Teaching page**: only lists McGill-affiliated courses (filtered by
  `"McGill" in location_string` in `build_teaching()`). Non-McGill courses
  in `TEACHING_COURSES` are intentionally filtered out, not deleted from the
  data, in case that's ever reversed.
- **Public Lectures page** (`talks.html`, nav label "Public Lectures", not
  "Talks"): a plain link list (`PUBLIC_LECTURES` in `data.py`), not the old
  invited-talks/conference-presentations tables. Only add links the site
  owner explicitly provides.
- **Media page**: appearance titles link out to their source URL when one
  is known (`MEDIA` tuples carry a 4th `url_or_None` field).
- **Footer**: `PARAMS_MYSITE_CREDIT` and `PARAMS_MYSITE_DISCOVERY` in
  `build.py` control a footer credit link and an invisible
  `<meta name="mysite-directory">` marker — these reflect the site owner's
  own opt-in from the original intake form. Leave both `True` unless told
  otherwise.

## Workflow notes

- After any change, run `python3 build.py` and spot-check the regenerated
  HTML (e.g. with Playwright/Chromium screenshots at both a desktop width
  like 1280px and a mobile width like 390px) before committing — this site
  has gone through many rounds of pixel-level layout feedback, so visual
  regressions matter.
- Commit and push directly to `main` when asked to publish — there is no PR
  review workflow for this repo; the owner reviews visually via a preview
  before approving, and finalizes by asking for it to be pushed.
- Prefer minimal, targeted edits to `data.py`/`build.py`/`style.css` over
  restructuring — the current structure reflects many specific, deliberate
  choices made over several rounds of feedback (see above), not defaults
  that should be "improved" unprompted.
